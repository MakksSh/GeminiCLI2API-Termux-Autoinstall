# GeminiCLI2API Termux Autoinstall 🧩🐧

Авто-установщик `geminicli2api` под Android/Termux:

* Сам обновит пакеты, поставит зависимости, скачает репозиторий, настроит `.env`, установит Python-зависимости
* Запоминает, на каком шаге остановился (можно просто перезапустить)
* Логирует всё в консоль и в файл

## Что нужно заранее

1. Установи **Termux** (обязательно из F-Droid https://f-droid.org/)
2. Прочитай гайд от Фрэнки https://t.me/AIandDrama/129 В нём нас интересуют разделы со вступлением, "Подготовка аккаунта" и "Как подключить". Раздел установки на Android пропускаем, скрипт сделает всё за вас.
3. У тебя должен быть **Google Cloud Project** (нужен его АЙДИ!)
4. БОНУС! Как получить доступ к Gemini 3 описано здесь: https://t.me/AIandDrama/151 Это можно сделать как до установки `geminicli2api` так и после.

> Найти ID проекта можно в Cloud Console: https://console.cloud.google.com/home/dashboard Убедись, что сверху слева выбран нужный проект. Плашка Project info -> Project ID


## Установка одной командой 

Вставь в Termux:

```bash
cd ~ && curl -fsSL "https://raw.githubusercontent.com/MakksSh/GeminiCLI2API-Termux-Autoinstall/refs/heads/main/cli2api.sh" -o cli2api.sh && chmod +x cli2api.sh && ./cli2api.sh
```

Скрипт спросит только `GOOGLE_CLOUD_PROJECT`

Можно сразу передать ID проекта аргументом (без вопросов):

```bash
cd ~ && curl -fsSL "https://raw.githubusercontent.com/MakksSh/GeminiCLI2API-Termux-Autoinstall/refs/heads/main/cli2api.sh" -o cli2api.sh && chmod +x cli2api.sh && ./cli2api.sh gen-lang-client-1234567890
```

Айди в конце замени на свой айди проекта

## Как запускать в следующий раз

Самый ленивый вариант:

```bash
./cli2api.sh
```

Он быстро пропустит шаги и снова запустит `run.py`

## Быстрая смена аккаунта/проекта Google

Скрипт имеет функционал быстрой замены проекта в одну команду. Он автоматически обновляет `.env` файл и сбрасывает авторизацию для входа под новым аккаунтом.

```bash
./cli2api.sh id
```

Вместо id в конце команды подставь свой айди проекта! После запуска пройди авторизацию в гугл по ссылке, которая появится в Termux.

## Что делает скрипт

Скрипт по шагам:

1. Обновляет все пакеты termux `pkg update && pkg upgrade`
2. Ставит нужные пакеты (python/rust/git/nodejs и т.д.)
3. Клонирует/обновляет репозиторий `geminicli2api`
4. Чинит `requirements.txt` (гарантирует `pydantic<2.0`)
5. Создаёт/обновляет `.env` и прописывает туда `GOOGLE_CLOUD_PROJECT`
6. Ставит Python-зависимости (в `.venv`, чтобы не засирать систему пакетами)
7. Запускает `python run.py` (cli остаётся работать в этой сессии)

## ВАЖНО: как правильно запускать Таверну рядом

* Сессию Termux, где запустился `geminicli2api`, не закрывай (это “сервер”)
* Открой вторую сессию Termux (или вкладку) и запускай там таверну. Для этого надо потянуть слева направо от края экрана (появится меню сессий) и нажать New Session

Остановить `geminicli2api` можно `Ctrl+C`. Ткнуть в экран, чтобы появилась клавиатура, переключиться на английскую раскладку, нажать кнопку `CTRL` над клавиатурой, затем букву C на клавиатуре

## Если что-то упало — что делать?

Ничего умного. Просто запускай скрипт ещё раз:

```bash
cd ~ && ./cli2api.sh
```

Он не начнёт заново, а продолжит с последнего завершённого шага

Если ошибка повторяется - присылай скриншоты с ошибкой в чатик канала Фрэнки (https://t.me/AIandDrama) или мне в личку. Контакты есть внизу

## Если скрипт видит папку `~/geminicli2api`

При первом запуске, если папка уже существует, он спросит:

**“Сделать полную переустановку (y/n)?”**

* `y` → удалит папку, всё обновит и поставит заново
* `n` → попробует “починить по месту” и продолжить

Введи соответствующую букву и нажми Enter (перенос строки) на клавиатуре

## Где логи и “память прогресса”

Скрипт хранит всё тут:

* Логи: `~/.geminicli2api-installer/install.log`
* Состояние (какой шаг выполнен): `~/.geminicli2api-installer/state.env`

## Как “сбросить всё и начать с нуля” (жёсткий ресет)

Если что-то пошло не так или ты хочешь начать установку с чистого листа, используй аргумент `--reset`:

```bash
cd ~ && ./cli2api.sh --reset
```

Скрипт предупредит о полном удалении данных, очистит папки `~/geminicli2api` и `~/.geminicli2api-installer`, после чего сразу начнет установку заново.

Можно комбинировать с ID проекта:
```bash
cd ~ && ./cli2api.sh --reset gen-lang-client-1234567890
```
